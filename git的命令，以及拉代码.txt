
问开发人员是基于那个分支进行开发的！


  将主分支代码 克隆下来后 git pull  一下  加origin (分支名) 更好


   如果不是主分支就进行切换   git checkout （切换的分支名字）

  
  在他说的那个分支上开发，就在那个分支上新建一个分支   git branch (分支名)
		查看分支名字     git branch -a

	建好分支后    进行切换 到自己的分支上进行修改，开发！

	修改完后 git status  可以查看自己修改过的文件！

		git add .
                git commit -m '修改'
		git push origin  (自己的分支名)
		
	然后切换到测试分支上，   
          git checkout (测试分支)

	在测试分支上再进行合并   git  merge （自己的分支）
	
	大公司的测试分支有几个 ，就是切换然后合并就行

	测试说没问题就 ok

	测试说有问题  就切换那个分支进行修改


	分支不能瞎命名

	vs cont 安装一个 gitles的编辑可以看到别人修改的记录


	分支爱出现问题  就是和别人修改的分支一样，会发生冲突，这时可以问是用你的还是用我的，也可以合并


	
    还有一个情景 你不想提交， 然后又要切换别的分支

		git stash  临时保存代码


         然后改你所要的需求


	git stash pop 拉下来
